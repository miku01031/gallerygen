## First Release Notes Draft

GalleryGen `v0.1.0` is the first public MVP release.

This release introduces the core end-to-end workflow for turning local images into a static gallery website directly in the browser.

### Highlights

- Multi-image upload with drag and drop support
- Non-image filtering during import
- Local image preview generation in the browser
- Editable gallery title and description
- Live default gallery template preview
- Static export bundle generation
- Browser-side ZIP download with HTML, CSS, and image assets

### Notes

- This release is intentionally focused on the MVP workflow.
- There is currently one template theme: `default`.
- Persistence, advanced metadata editing, and additional template options are not included yet.

### V2EX / Reddit Style

I have been building a small tool called GalleryGen and just published the first MVP.

The idea is simple: take a batch of local images, preview them as a clean gallery site, then export a ZIP of static files you can host anywhere that serves HTML/CSS/images.

Current MVP includes:

- multi-image upload
- drag and drop
- title and description editing
- live gallery preview
- ZIP export in the browser

Still early, but the full core loop is working. Feedback on usability, output quality, and roadmap direction would be very welcome.
